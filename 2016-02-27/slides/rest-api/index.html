CRUD

Verbo Azione URI Risposta Descrizione
GET Read /posts 200 OK Elenco posts
GET Read /posts/{id} 200 OK Dettaglio post
POST Create /posts 200 OK Post creato
PUT Update /posts/{id} 200 OK Post aggiornato
DELETE Delete /posts/{id} 204 No Content Post eliminato

Nel caso in cui il DELETE contenesse una risposta, allora utilizza il codice 200 Nel caso in cui il POST contenesse solo il link all'entità, allora utilizza il codice 201 Created
(Location Header)

RELAZIONI

Con URL lineari sarà intuitivo accedere alle relazioni


GET    /posts/1234/comments
GET    /posts/1234/comments/1

POST   /posts/1234/comments

PUT    /posts/1234/comments/1
DELETE /posts/1234/comments/1

FORMATI

Affidati agli Header per definire il formato di risposta


/posts?format=json


Accept: application/json
Viene gestito automaticamente da Laravel per formattare le risposte (es. Unauthorized)

VERSIONING

È una buona idea utilizzare una versione per le tue API specificandola nelle URL o nell'Header

URL


/api/v1/posts
/api/v2/posts

Vendor Tree


Accept: application/vnd.github[.version].param[+json]
https://tools.ietf.org/html/rfc6838#section-3.2

FILTRO E ORDINAMENTO

Utilizza le querystring per filtrare e ordinare i dati, ma ricorda ... non complicare le cose.

Filtro


/posts?before=20160123&author=billmn

Ordinamento


/posts?sort=+author,-created_at

TRANSFORMATION LAYER

Funge da layer tra i dati letti, es. da database, e le tue API.

Questo consente di mantenere consistente l'output anche se la struttura delle tabelle viene modificata.

Un altro vantaggio è quello di poter formattare alcuni valori delle API come (es. le date in formato ISO8601) senza intaccare il resto dell'applicazione.

http://fractal.thephpleague.com
